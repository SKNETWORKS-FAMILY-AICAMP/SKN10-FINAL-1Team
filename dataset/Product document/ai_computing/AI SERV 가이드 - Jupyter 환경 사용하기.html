AI SERV 가이드 - Jupyter 환경 사용하기

Jupyter Notebook 사용하기

실행 중인 컨테이너를 사용하고 관리하는 법에 관해 살펴보겠습니다.
메인 탭에서 컨테이너 선택 후 ‘앱 실행’ 버튼을 클릭하면 다음 이미지와 같이 앱 런처가 뜨면서 해당 컨테이너가 지원하는 몇 가지 앱 서비스가 뜨게 됩니다.





Jupyter Notebook을 클릭해봅시다.
새로운 창이 뜨면서 Jupyter Notebook이 실행되는 것을 확인할 수 있습니다. 이 Notebook은 실행 중인 컨테이너 내부에서 생성된 것으로, 별다른 설정 없이 버튼 클릭만으로 손쉽게 사용할 수 있습니다. 또한 컨테이너가 기본적으로 제공하는 언어 환경 및 라이브러리를 그대로 활용할 수 있어 별도의 패키지 설치 과정이 필요 없습니다. 자세한 Jupyter Notebookk 사용 법은 공식 문서 등을 참고하시기 바랍니다.
Notebook의 파일 탐색기에서 id_container 파일은 private SSH Key를 담고 있습니다. 필요할 경우 다운로드 하여 컨테이너로의 SSH/SFTP 접속에 이용할 수 있습니다.


데이터 업로드하기

볼륨 관리 가이드에서 설명한 바와 같이, AI SERV는 사용자의 파일을 보존할 수 있도록 별도의 볼륨을 지원합니다.
컨테이너의 기본 폴더는 컨테이너가 삭제되면 같이 삭제되므로, 보존해야 하는 파일이 있을 경우 Data 폴더에 저장하는 것이 좋습니다.

우측 상단의 UPLOAD 버튼을 클릭하면 파일 탐색기 창이 뜨며, 단일 파일 혹은 여러 파일을 동시 선택하여 데이터를 업로드할 수 있습니다.
Jupyter Notebook 환경에서 폴더 단위의 데이터 업로드는 지원하지 않습니다. 폴더를 압축한 후 압축파일을 업로드하는 방법을 활용할 수 있습니다.


ipynb 활용하기

우측 상단의 NEW 버튼을 클릭한 후 AI SERV용 Notebook을 선택하면 새로운 코드를 입력할 수 있는 ipynb 창이 뜹니다.



이 창에서 세션 환경에 맞는 코드를 입력하고 실행해볼 수 있습니다. 코드는 AI SERV 서버를 구성하는 노드 중 연산 세션이 실제로 생성된 노드에서 실행이 되며, 로컬 머신에서는 별도 환경을 구성할 필요가 없습니다.



창을 닫으면 Notebook 파일 탐색기에 방금 작업한 Untitled.ipynb 파일이 생성되어 있는 것을 확인할 수 있습니다. 여기에 생성된 파일은 컨테이너를 삭제할 경우 같이 삭제되는 것에 주의하십시오.
