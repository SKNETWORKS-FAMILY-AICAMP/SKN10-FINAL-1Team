상품 개요

상품 이용 전, 주의 사항

  • Beta 서비스 기간 동안, 상용 시스템 구축에 해당 상품의 이용을 권장하지 않습니다.
  • 테스트, 개발 시스템 구축 목적의 이용을 권고 드립니다.
  • Redis사의 라이선스 정책 변경('24.3.20)으로 인하여 Opensource SW 마지막 버전인 7.2 와 호환됩니다.
  • Redis 7.2 EOL '25.8.31 이후 Opensource SW 자체의 업데이트가 제공되지 않습니다.

    • 목적/용도

    DBaaS for Cache는 In-Memory DB를 편리하게 배포하고 운영할 수 있는 서비스입니다.

    Master/Replica, Cluster 구조를 자동으로 구성하여 고 가용성과 서비스 연속성을 얻을 수 있습니다.
    UI로 손쉽게 구조 변경이 가능하며, 실수를 예방하고 안정적으로 DB 서비스를 이용할 수 있는 다양한 관리 기능을 제공합니다.

    서비스 구조/원리

    Server 상품과 동일하게 VM 형태의 DB Instance로 고객 네트워크 내에 생성됩니다. DB Instance에 대한 모든 생성/변경/삭제/기타 관리 사항은 서비스 콘솔에서 이용 가능합니다.



    제약 사항

  • DB Instance Spec. 지원 목록



  • Cache용으로 사용가능한 메모리는 계산 방식은 아래와 같습니다.
    (e.g.) Standalone, Master/Replica : VM Spec 내 Memory 크기의 ½ (4vCore 32GB로 선택 시, 32/2=16GB Cache 이용가능 )
    Cluster : VM Spec 내 Memory 크기의 ½ Shard개수 (4vCore 4GB, Shard 3 선택 시, 4/2*3= 6GB Cache 이용가능)


  • DB Server Type



    • Master/Replica 구조는 Master 1대, Replica1대, Sentinel(1vCore 1GB) 1대로 총 3대로 고정됩니다.

  • 서비스 지원 범위

    • 현재 DBaaS for Cache 서비스는 Openstack 사용환경을 지원 합니다.

    • 지원 대상
      • 민간 클라우드 : D1플랫폼 DX-M1, DX-Central 존
      • 공공 클라우드 : D1플랫폼 DX-G, DX-G-YS존
