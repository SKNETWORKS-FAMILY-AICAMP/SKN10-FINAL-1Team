상품 개요

목적/용도

DBaaS for MySQL 8은 최신 MySQL 데이터베이스를 편리하게 배포하고 운영할 수 있는 서비스입니다.

특히, MySQL Native한 클러스터 구조를 자동으로 구성하여 고 가용성과 서비스 연속성을 보장 받을 수 있습니다.
UI로 손쉽게 MySQL 구조 변경이 가능하며, 실수를 예방하고 안정적으로 DB 서비스를 이용할 수 있는 다양한 관리 기능을 제공합니다.

서비스 특장점

ㅇ UI 기반 설정 : UI 기반의 편리한 DB 구조 변경(Standalone ←→ Cluster)
ㅇ 고 가용성 : MySQL Cluster의 Single/Multi Primary 구조로 Read/Write 부하 분산 기능 제공
ㅇ 안정성 : MySQL 장애 발생 시, 자동 Fail Over/Fail Back 수행
ㅇ 파라미터 그룹 관리 : 복잡한 설정을 간단히 관리하는 파라미터 그룹 관리 기능
ㅇ 백업 기능 : 데이터 유실을 최소화 하기 위한 다양한 Backup 기능 제공
ㅇ 모니터링 : 모니터링, 알람 기능 제공


서비스 구조/원리

DBaaS for MySQL 8은 Server 상품과 동일하게 VM 형태의 DB Instance로 고객 네트워크 내에 생성됩니다.
DB Instance에 대한 모든 생성/변경/삭제/기타 관리 사항은 DBaaS for MySQL 8 서비스 콘솔에서 이용 가능합니다.




제약 사항

  • DBaaS for MySQL 8 DB Instance Spec. 지원 목록



  • DB Server Type



    • * 단, Cluster 구조 생성 시 MySQL Router용 VM(1vCore,1GB Mem.) 2대가 추가 생성됩니다. MySQL Router는 Application과 DB Cluster 간의 연결을 설정해 주며, 이중화(Active-Active) 구성됩니다. MySQL Router용 VM도 과금됩니다.
    * Cluster 구조의 경우 서버는 홀수개로 구성하는 것을 MySQL에서 권장합니다. (최소 3대~최대9대)
    * MySQL 8.0.29 버전의 자체 버그로 인해 9대로 서버 구성하는 것이 제한됩니다. (9대의 서버로 구성한 경우, Cluster의 Multi-Primary 구조는 서버간 auto_increment 값의 충돌이 발생될 수 있음)
    * MySQL 8.0.33 이상 버전부터 9대로 서버 구성이 가능합니다.
    * Stand Alone 구조는 테스트 용도의 사용을 권고 드립니다.
    * 일반적인 OLTP환경(Read7 : Write3)하에서 Cluster의 Single Primary 구조 사용을 권고 드립니다.
    * Write이 많은 환경에서 고사양의 Multi Primary 구조 사용을 권고 드립니다.
