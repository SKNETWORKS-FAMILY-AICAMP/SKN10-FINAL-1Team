AI SERV 가이드 - 컨테이너 관리

새로운 컨테이너 시작하기

볼륨 생성을 완료한 후 ‘컨테이너 생성’을 클릭하여 컨테이너 생성 페이지로 이동합니다.
다음과 같은 화면으로 연결됩니다.





  • Name : 생성할 컨테이너의 이름을 지정할 수 있습니다. 컨테이너 이름은 4-64자의 알파벳 또는 숫자만 받아들이며, 공백은 허용되지 않습니다.
  • 자원 그룹 : 컨테이너를 생성할 자원 그룹을 지정합니다. 자원 그룹은 각 사용자가 접근할 수 있는 호스트 서버를 묶은 단위이며, 주로 같은 종류의 GPU 자원을 보유하고 있는 서버들을 하나의 자원 그룹으로 설정합니다.
  • Environments : TensorFlow, PyTorch 등과 같은 컨테이너의 기본 환경을 지정합니다. TensorFlow를 선택하면 컨테이너에서 TensorFlow 라이브러리를 사용할 수 있습니다. 다른 환경을 선택하면 해당 환경이 기본적으로 설치된 컨테이너를 생성하게 됩니다.
  • Version : 환경의 버전을 선택합니다. 예를 들어, PyTorch 환경에서는 1.14 등과 같은 버전을 선택할 수 있습니다.
  • Resource Allocation : 컨테이너에 할당할 CPU, 메모리, GPU 등의 자원 량을 사전에 정의해 둔 템플릿입니다.

    • 이전 생성해 놓은 볼륨은 컨테이너에 자동으로 마운트됩니다.
    ‘생성하기’ 버튼을 클릭하여 진행합니다.





    메인 탭에서 새로운 컨테이너가 생성되는 것을 확인합니다.



    메인 탭에서 현재 실행 중인 컨테이너에 대한 정보를 확인할 수 있습니다.
    컨테이너 별로 상태, 할당 및 사용 자원량, 컨테이너 시작 시간 등의 정보를 확인할 수 있습니다.

    NVIDIA A100 자원 그룹 사전 정의 상품
    해당 템플릿 이외의 자원을 할당하고 싶다면, 관리자에게 문의하십시오.



    컨테이너 관리

    AI SERV는 PENDING, RUNNING, TERMINATING 등 컨테이너에 대한 상세 상태 정보를 제공합니다.
    PENDING 컨테이너의 경우 현재 전체 자원 pool에서 할당 가능한 자원이 없어 대기열에 머물러 있는 상태입니다.



    컨테이너 삭제

    특정 컨테이너를 삭제하기 위해서는 컨테이너 선택 후 ‘컨테이너 삭제’ 버튼을 누르면 됩니다.
    컨테이너 내부 폴더의 데이터는 컨테이너가 종료되는 순간 같이 삭제되므로, 계속 유지해야 하는 데이터라면 마운트한 볼륨에 옮기거나 처음부터 볼륨에 업로드하는 것을 권합니다.
