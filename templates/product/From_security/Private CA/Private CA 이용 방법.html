Private CA 이용 방법

CA 관리

CA 관리 화면에서 CA를 신규로 생성하고 관리할 수 있습니다. 또한 생성된 CA를 통해 사설 인증서를 발급할 수 있습니다.
CA 관리 화면에서는 CA 생성, 사설 인증서 발급, CA 인증서 보기 등 CA관리를 위한 다양한 기능을 제공합니다.



CA 관리 화면은 다음 3개의 영역으로 구성됩니다.

  1. 상단 메뉴 바 영역
  2. CA 관리 화면의 좌측은 CA의 목록 조회 화면이며, Root CA-Sub CA간 계층 트리 형태로 표시됩니다.
  3. CA 관리 화면 우측 박스는 좌측 CA 목록에서 선택한 CA의 세부 정보가 표시됩니다.

CA 상세 정보의 내용은 아래와 같습니다.

  • 일반 이름(Common Name): CA 일반 이름
  • CA 상태: 활성화, 비활성화, 등록 대기 등 CA 운영 상태를 표시
  • 유효기간: CA 유효 기간의 시작 일시, 종료 일시
  • CA Tag: CA 생성 시 자동 할당되는 고유 식별자
  • 발급 인증서: 발급된 사설 인증서 수
  • CRL: 인증서 폐기 목록(Certificate Revocation List) 배포 지점으로 인증서 폐기 목록 제공을 위한 URL
  • OCSP: 온라인 인증서 상태 프로토콜(Online Certificate Status Protocol) 제공 URL
  • 키 알고리즘: CA의 공개키 암호화 알고리즘
  • 서명 알고리즘: CA 서명 암호화 알고리즘
  • 메모

CA 생성

CA관리 화면에서 상단 메뉴 바의 CA 생성 버튼을 클릭하여 CA를 생성 할 수 있습니다.
CA 생성은 1. 일반 정보, 2. 상세 설정 정보, 3. 신청 정보 확인의 3단계를 통해 이루어집니다.

1. 일반 정보
CA 신규 생성을 위한 일반 정보 항목을 입력 후 화면 하단의 다음 버튼을 클릭합니다.



  • CA 타입: 생성할 CA의 유형으로 Root CA 또는 Sub CA를 선택할 수 있습니다.
  • 이름: 생성할 CA 이름을 입력합니다.
  • 상위 CA: CA 타입이 Sub CA일 때 노출됩니다.
    • - 외부에서 발급: Private CA가 아닌 외부 CA에서 Sub CA 인증서를 발급받고 해당 인증서를 가져와서 등록하는 방식일 때 이용
    • - 기존 CA: Private CA에서 생성/관리하는 CA를 사용하여 Sub CA를 생성하는 경우
  • 유효기간: 생성할 CA의 유효 기간을 일 단위로 입력합니다. (최대 3,650일)
  • 키 타입: CA의 공개키 암호화 알고리즘을 선택합니다.
  • 메모: CA 관리를 위한 간단한 메모를 입력합니다.


2. 상세 설정 정보
CA 주체 정보와 주소 정보를 입력합니다.



  • 일반 이름: 생성할 CA의 일반 이름을 입력합니다.
  • 조직: 소속 회사 혹은 기관 정보를 입력합니다.
  • 부서: 소속 부서 정보를 입력합니다.
  • 국가: KR(대한민국)로 지정되어 있습니다.
  • 도(State/Province): 주/도 또는 지방의 이름 (영문 입력)
  • 시: 도시 (영문 입력)
  • 상세 주소: 시, 도 외 나머지 주소 정보 (영문 입력)


3. 신청 정보 확인
CA 생성을 위해 입력된 일반 정보, 상세 설정 정보의 내용을 다시 검토하고 확인 버튼 클릭 시 CA가 생성됩니다.



CA 생성이 완료되면 CA관리 화면 좌측 트리 구조에 생성된 CA가 표시됩니다

  • Sub CA는 상위 CA보다 유효 기간을 초과하여 생성할 수 없으며, 상위 CA보다 유효 기간을 초과하여 생성 시 상위 CA의 유효 기간과 동일하게 자동 설정됩니다.


    • 외부 CA 인증서 등록

    Sub CA를 생성 시 상위 CA를 “외부에서 발급”을 선택하여 직접 서명 방식으로 CA 인증서를 외부에서 발급 받을 수 있습니다.
    CA 관리 화면 > CA 생성 버튼 클릭 > 상위 CA 항목에서 외부에서 발급 선택 후 CA를 생성합니다.


    외부 발급 선택 시 아래 화면과 같이 CA 생성 이후 CA는 등록 대기 상태가 되며, 외부에서 발급 받은 인증서를 등록 후에 CA가 활성화 상태로 전환 됩니다.

  • CA관리 화면 상단 메뉴에서 CSR 다운로드 기능을 이용하여 CSR 파일을 다운로드 받습니다
  • 해당 CSR을 이용하여 외부에서 자체 서명한 인증서를 생성 후 상단 메뉴 인증서 등록 기능을 이용해 인증서를 등록하시면 됩니다.


    • CA 인증서 보기

    CA관리 화면 상단의 CA 인증서 보기 버튼을 클릭하여 CA 인증서를 확인할 수 있습니다.
    아래와 같이 CA 인증서의 세부 정보를 조회 가능하며 인증서 다운로드가 가능합니다.



  • 인증서 본문 다운로드 : PEM 타입의 인증서를 다운로드 할 수 있습니다
  • 인증서 체인 다운로드 : PEM 타입의 인증서 체인을 다운로드 할 수 있습니다.

    • CA 활성화/비활성화

    운영중인 CA는 필요에 따라 활성화 -→ 비활성화, 비활성화 -→ 활성화로 상태를 변경할 수 있습니다.
    상단 메뉴 바 의 활성화, 비활성화 메뉴를 통해 CA의 상태 변경이 가능합니다



  • CA의 상태를 비활성화 상태로 변경하게 되면 사설 인증서를 발급하는 등 전반적인 기능 사용이 중단됩니다.(조회만 가능)
  • 비활성화 상태에서 활성화 기능을 이용하면 다시 CA는 활성화 상태로 변경이 가능합니다.

    • CA 삭제

    운영 중인 CA가 더 이상 불필요할 경우 CA 삭제 요청 기능을 통해 삭제가 가능합니다.
    CA 삭제 요청 시 CA는 즉시 삭제되지 않고 삭제 예정 상태로 변경되며 72시간 대기 이후 삭제됩니다. 삭제 예정 상태로 변경 후 대기 시간이 불필요할 경우
    즉시 삭제 기능을 통해 즉각적인 CA의 삭제 완료가 가능합니다.

    CA 삭제 방법은 CA관리 화면 > 상단 메뉴의 삭제 요청 기능 선택하시면 됩니다.
    CA 삭제 요청시 아래와 같이 확인 창이 나타나며, CA 삭제 확인 항목에 삭제 대상 CA 이름을 입력하고 확인 버튼을 클릭하면 CA는 삭제 예정 상태로 변경됩니다.


  • CA 삭제 요청 이후 72시간 대기 시간 이후 CA가 자동 삭제되며 CA 하위의 Sub CA와 CA 개인 키등 모든 정보가 삭제되어 복구 되지 않습니다.
  • CA 삭제 예정 상태에서 즉시 삭제를 통해 CA 정보를 즉각적으로 삭제하거나 또는 삭제 취소 기능을 이용해 활성 상태로 변경이 가능합니다.
  • CA 삭제 요청 이후 즉시 삭제 시에도 동일하게 CA 정보는 즉시 삭제되며 삭제된 CA는 복구 되지 않습니다.
  • CA 삭제 시에는 CA의 활성화 여부, 만료 여부 등을 확인하지 않습니다.

    • CA 상태 관리

    Private CA에서 생성한 CA는 활성, 비활성, 등록 대기등 인증 기관의 상태가 변경될 수 있으며, 상태에 따라 사용할 수 있는 기능에는 차이가 있습니다.

    • 활성: CA의 모든 기능을 사용할 수 있습니다.
    • 비활성: 조회 기능을 제외한 기능이 정지된 상태입니다.
    • 등록 대기: Private CA가 아닌 외부 CA에서 발급 받은 인증서를 등록하기 전 상태입니다.
    • 만료: CA 인증서의 유효 기간이 만료된 상태입니다. 만료된 CA는 더 이상 사용할 수 없습니다.
    • 삭제 예정: 사용자에 의해 CA 삭제 요청된 된 상태입니다. 72시간 대기 후 자동 삭제됩니다.
    • 완전 삭제: Private CA에서 해당 CA가 삭제된 상태로 삭제된 CA는 더 이상 데이터 조회 등 기능 지원이 되지 않습니다.


    아래 표는 CA의 각 운영 상태 따른 기능 사용 가능 여부를 나타냅니다.


    사설 인증서 발급

    생성한 CA를 통해 사설 인증서를 발급할 수 있습니다.
    CA 관리 화면의 좌측 트리에서 CA를 선택 후 상단 메뉴바의 사설 인증서 발급 버튼을 클릭합니다.

    사설 인증서를 발급하기 위해서는 CA생성 시와 유사하게 1) 기본정보, 2) 주체 정보, 3) 주소 정보, 4) SANs 정보를 입력합니다.
    발급이 성공하면 아래와 같은 팝업 화면이 표시되며 인증서 발급 결과 화면에서 인증서, 개인 키, 인증서 체인을 다운로드 받을 수 있습니다.


    • 발급한 사설 인증서의 개인 키는 발급 결과 화면에서만 확인 가능하니 인증서의 개인 키는 다운로드 후 안전하게 관리해 주시기 바랍니다.
    • Private CA는 발급한 사설 인증서의 개인 키를 별도 관리하지 않으며, 이후에 개인 키의 재 확인은 지원되지 않습니다.


    발급 인증서

    CA에서 발급한 사설 인증서 내역을 확인할 수 있습니다.
    상단 메뉴 바에서 발급 인증서를 선택하면 아래와 같이 해당 CA에서 발급한 사설 인증서 목록이 조회 되며 인증서 상세 보기 기능을 통해 인증서 세부 정보 조회 및 인증서 다운로드가 가능합니다.


    사설 인증서 폐기

    발급한 사설 인증서는 만료되지 않은 상태에서 필요에 따라 폐기할 수 있습니다.
    인증서 폐기는 발급 인증서 화면 > 인증서 목록에서 인증서 선택 > 상단 메뉴 바의 폐기 요청 버튼 클릭을 통해 가능합니다.

  • 삭제된 인증서는 인증서 폐기 목록(CRL)에 등록됩니다.
  • 삭제된 인증서는 발급 인증서 목록에서 삭제되지는 않으며 상태만 폐기로 변경됩니다.


    • 인증서 유효성 검증 지원(CRL, OCSP)

    Privcate CA는 인증서 유효성 검증을 위한 CRL, OCSP프로토콜을 기본적으로 지원합니다.
    CA의 CRL 배포 지점 , OCSP URL은 사설 인증서 발급 시 인증서에 포함되어 명시됩니다.

    CRL

    CRL(인증서 해지 목록, Certificate Revocation List)은 인증 기관에서 제공 됩니다. 인증서 해지 목록에는 무효화되거나 해지 된 인증서의 일련 번호가 포함되어 있습니다.
    서버에 액세스 요청이 수신 되면 서버는 CRL을 기반으로 액세스를 허용하거나 거부합니다.

    CRL은 Private CA에서 기본 제공되며 CA 생성 후 CA 관리 화면 우측 상세 정보 에 해당 URL이 표기됩니다

  • CRL 배포 지점은 별도 생성할 필요가 없으며 CA별로 자동 생성 및 지정됩니다.


    • 생성된 CRL 파일을 확인하거나 관련 설정을 변경하는 방법은 다음과 같습니다
    CA 관리 > 상단 메뉴 바 페지목록 관리 기능 선택 > 아래와 같은 팝업 화면에서 CRL 선택 후 다운로드 버튼 클릭으로 가능합니다.


  • 인증서 폐기 목록은 매일 갱신 되며 필요에 따라 유효 기간 설정 변경도 가능합니다.
  • 인증서 폐기 목록은 매일 갱신 되지만 필요에 따라 CRL 갱신 기능을 통해 즉각적으로 현행화 할 수 있습니다.


    • OCSP

    OCSP(온라인 인증서 상태 프로토콜, Online Certificate Status Protocol)를 생성하여 인증서의 폐지 여부를 실시간으로 제공할 수 있습니다.
    OCSP를 생성하면 이후 발급 되는 사설 인증서에는 실시간 인증서 폐지 여부 확인을 위한 OCSP URL이 추가됩니다.
    (사설 인증서의 기관 정보 액세스 항목에서 온라인 인증서 상태 프로토콜 항목 확인)

    OCSP 생성은 CA 관리 > 우측 CA 상세 화면에서 OCSP 항목 우측의 생성 버튼을 클릭하면 OCSP URL이 표기됩니다.
