Q. 웹 방화벽을 이용하기 위한 방화벽 설정은 어떻게 하나요?

웹 방화벽(WAF)을 사용하기 위해서는 아래 방화벽 설정이 필수적으로 수행되어야 서비스를 이용할 수 있습니다.


1. 포탈 서버에서 웹 방화벽(WAF)으로의 방화벽 설정
source IP: 14.63.254.11, 95
destination IP: 웹 방화벽(WAF)의 공인아이피
Port: 5950 ~ 5999 (이중 WAF에 등록된 포트)

2. 웹 방화벽(WAF)에서 인증 서버로의 방화벽 설정
source IP: 웹 방화벽(WAF)의 공인아이피
destination IP: 218.145.29.166, 218.145.29.168, 218.145.29.101
Port: TCP(443)

Q. 웹 방화벽을 최신버전으로 업데이트하려면 어떻게 해야 하나요? 업데이트 시 서비스 중단이 발생하나요?

  • 업데이트 상세 방법은 시스템 업데이트 가이드를 참고하시기를 바랍니다.
  • 업데이트 시 약 15분간 안정적인 서비스 제공이 어려움으로 DNS의 설정을 WAF에서 웹서버로 변경하거나, LB를 사용하는 경우, LB에서 웹서버로 트래픽이 넘어가도록 설정 후 업데이트하시기 바랍니다.
  • 시스템의 안정적 운용을 위해서 최소 3개월에 한 번씩은 최신버전으로 업데이트를 권고합니다.

Q. 웹 방화벽 신청 후 팝업 및 Email로 안내 받은 패스워드로 관리도구 접속이 안 됩니다.

  • 웹 방화벽은 관리를 위해 관리도구와 SSH 터미널 두 가지를 제공합니다.
  • 신청 시 팝업 및 Email로 안내 받은 패스워드는 SSH 터미널 패스워드이며, 관리도구의 경우, 초기 패스워드가 고정되어 있으므로 최초 로그인 후 비밀번호를 변경하시면 됩니다.
  • 관리도구 초기 패스워드는 Tech Center로 문의하시기 바랍니다.

Q. 계정과 패스워드를 정확히 입력했는데도 관리도구 접속이 안 됩니다.

  • 웹 방화벽 생성 시, 입력한 포트 정보를 정확히 입력해야 합니다. 클라우드 콘솔의 포트 포워딩 설정 정보와 입력한 포트 정보가 일치하는지 확인이 필요합니다.
  • 관리콘솔 접속 시 https://웹방화벽IP:ConsolePort를 입력해 주시면 됩니다.

Q. 관리도구 패스워드 분실 시 어떻게 해야 하나요?

  • 관리도구 패스워드 분실 시 Tech Center로 문의하시기 바랍니다.

Q. 탐지된 웹 로그 기능 조회 기능이 제공 되나요?

  • 기간별/출발지 IP/URL/보안 정책/룰 등 필요한 조건에 맞게 필터링하여 조회할 수 있습니다. (WAF 관리콘솔 내 탐지 로그 메뉴)

Q. 공격 탐지 결과를 보고서 형태로 받을 수 있나요?

  • 관리도구 접속 후 메뉴 탭에서 "보고서" 메뉴 선택 창에서 필요한 보고서 유형을 선택하여 이용할 수 있습니다.

Q. 정책 설정을 백업할 수 있나요?

  • 정책 백업 상세 방법은 시스템 업데이트 가이드를 참고하시기를 바랍니다.

Q. 탐지 로그의 Client IP가 실제 Client IP와 달라요.

  • 웹 방화벽 상단에 LB 등 Proxy 형태의 장비가 있을 경우, Proxy 장비 특성으로 인해 TCP의 출발지 IP가 Proxy 장비의 IP로 변경됩니다.
  • 이에 WAF에서 탐지 로그 생성 시 TCP의 출발지 IP를 참조하도록 되어 있음에 따라 Proxy IP가 기록됩니다.
  • 이런 Proxy 형태의 장비들은 Client IP를 보존하기 위해 X-Forwarded-For 헤더 내에 Client IP를 저장하고 이를 Request 메시지 헤더에 포함하여 전송합니다.
  • 웹 방화벽에서도 설정을 통해, Proxy 환경일 경우 해당 헤더를 참조하여 탐지 로그를 생성하도록 적용할 수 있습니다.

설정 방법
관리도구 → 보안 설정 → 정책 부가 설정
