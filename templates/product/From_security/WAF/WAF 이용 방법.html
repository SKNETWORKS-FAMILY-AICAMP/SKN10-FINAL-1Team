WAF 이용 방법

kt cloud 포탈에서 WAF 서비스 구성 및 환경 설정하는 방법을 설명합니다.


* 2018년 8월 31일 이전 M2존 생성 및 2019년 12월 12일 이전 Central-A존 생성 웹방화벽의 경우
웹방화벽 관리도구 - 4.0 Version을 참고하시기 바랍니다

웹 방화벽의 생성 및 삭제

WAF는 다음 순서로 웹 방화벽 신청 화면에서 서비스 구성을 합니다.
상품소개에서 보안 -> 콘솔에서 WAF 선택 -> 웹 방화벽 생성 버튼 클릭

1 단계 서비스 구성


  1. WAF가 생성되기를 원하는 위치를 선택합니다.
  2. WAF 이름을 입력합니다.
  3. Single 선택 시, WAF의 이중화를 구성하려면 Single을 2개 생성하여 구성합니다.
  4. 사양: 필요 트래픽에 따라 상품(Standard, Advanced, Premium)을 선택합니다.
        각 상품별 Throughput은 (350 / 600 / 1000)Mbps이므로, 서비스 최대 트래픽을 고려하여 상품 선택이 필요하며,
        안정적 서비스를 위해 반드시 이중화 구성을 권고합니다.
  5. WAF가 사용하는 3개의 포트에 대한 포트포워딩 설정을 합니다.
    • API: 사설 IP:5001과 매핑되는 포트
    • SSH: 사설 IP:22와 매핑되는 포트
    • Web Console: 사설 IP:5001과 매핑되는 포트
  6. 포트 포워딩 설정에 충돌이 없는지 포트 체크를 합니다.

※ 고객의 웹서비스 환경에 따라 상품별 최대 Throughput은 달라질 수 있습니다.

2 단계 신청 내역 확인

신청 내역을 확인하고 확인 버튼을 클릭 후 웹방화벽 서버가 생성 완료될 때까지 대기합니다.
3분 후 새로고침을 통해 상태 변경을 확인합니다. 완성 시 메일이 발송되어 메일 수신 여부로도 생성 여부 확인이 가능합니다.



서버 생성 및 구성이 완료되면 WAF 상태가 준비 중에서 사용 상태로 변경됩니다.
kt cloud email 계정의 메일을 확인하셔서 SSH 접속 패스워드를 확인하실 수 있습니다



  • 웹 방화벽 VM 의 삭제는 웹 방화벽 VM 이 정지된 상태에서 삭제버튼을 클릭하면 삭제 가능합니다.
  • 웹 방화벽 초기화 및 업데이트 정보를 가져올 수 있도록 3 개 IP (218.145.29.166, 218.145.29.168, 218.145.29.101)에 대한 방화벽 허용 설정이 필요 합니다.

    • WAF VM의 시작/정지, SSH 접속 비밀번호 변경

    시작/정지

    웹 방화벽 신청 시 자동으로 시작 상태가 됩니다. 특별한 사유에 의해 웹 방화벽을 재부팅하고자 할 때 다음과 같이 정지/시작 할 수 있습니다.

  • 웹 방화벽 선택 후, 정지 버튼을 클릭합니다.
  • 정지 클릭 후, 웹방화벽 상태가 정지 될 때까지 대기합니다.
  • 웹 방화벽 상태가 정지된 후, 다시 시작 버튼을 클릭합니다.




    • ※ 참고 내용
    6.0 버전의 경우 웹 방화벽 관리도구에서 Host 기준 Proxy 보호 대상 설정을 지원합니다.
    관리도구 → 네트워크 설정 → 보호 대상 서버 메뉴에서 내장된 매뉴얼 다중 Proxy 부분 참고하시기 바랍니다.

    SSH 접속 비밀번호 변경

    웹 방화벽은 주로 웹 기반의 GUI 관리콘솔을 사용하여 관리하며, 상세 점검 등 특별한 경우 SSH 접속을 하여 관리할 수도 있습니다.
    SSH 접속 비밀번호는 다음과 같은 절차로 변경할 수 있습니다.

  • 웹 방화벽을 선택합니다.
  • 웹 방화벽을 정지하고 우측 상단의 ... 위치에 마우스 커서 이동합니다.
  • 비밀번호 변경 클릭 및 팝업 창에서 확인 클릭 시, 랜덤하게 비밀번호 변경됩니다.




    • 웹 서버 구성 및 웹 사이트 구성

    웹서버 구성

    웹 방화벽과 웹 서버간의 서비스 구성작업을 실시 합니다.



    1. 구성하고자 하는 웹 방화벽을 선택 후 상단에 웹서버 버튼을 클릭합니다.
      웹 방화벽이 2개의 웹서버를 보호하는 경우를 가정하여 설명합니다.
    2. 보호할 웹서버를 선택합니다.
    3. 웹 서버의 웹서비스 포트를 입력합니다.
    4. 웹 방화벽의 Proxy 포트를 입력합니다.
      (트래픽은 웹 방화벽 IP:Proxy Port → 웹서버 IP:웹서버 Port로 전달됩니다.)
    5. 웹 방화벽의 이중화 구성의 경우 부하분산 처리를 로드밸런서를 선택해 줍니다.
      LB에 별도 설정할 필요 없이 여기서 입력되는 값 기준으로 LB와 WAF, 웹서버간 연결이 형성됩니다.
    6. 설정값 입력이 완료되었다면, 추가를 클릭하면 하단에 입력 정보가 반영됩니다.
    7. 입력값이 잘못되면 삭제 후 다시 입력할 수 있습니다.

    ※ 참고 내용
    6.0 버전의 경우 웹 방화벽 관리도구에서 Host 기준 Proxy 보호 대상 설정을 지원합니다.
    관리도구 → 네트워크 설정 → 보호 대상 서버 메뉴에서 내장된 매뉴얼 다중 Proxy 부분 참고하시기 바랍니다.

    웹 사이트 구성

    웹 사이트 구성은 웹 방화벽과 웹서버의 서비스 구성이 완료된 상태에서 웹서버를 보호하기 위해 웹 서비스의 HTTP/HTTPS 메시지에서 사용되는 IP 또는 URL을 이용하여 설정합니다.
    즉, 웹 방화벽에서 해당 웹서버로 연결되는 트래픽의 메시지 헤더 내용이 웹사이트 등록된 내용과 일치해야 하며, 일치하지 않을 경우는 웹 서비스가 차단됩니다.



    1. 웹 방화벽 서버를 클릭합니다.
    2. 웹사이트를 클릭합니다.
    3. 웹 방화벽이 보호해야 하는 URI 또는 IP를 등록합니다. 웹 방화벽에서는 HTTP(S)의 메시지를 점검하여
      외부 공격을 차단하기 때문에 HTTP(S)의 메시지 URI에 해당하는 부분을 웹사이트로 등록해야 합니다.
      (Client에서 입력되는 URI와 동일하게 입력)
    4. 웹사이트 포트를 입력합니다.
    5. 사이트를 보호할 보안정책을 선택합니다.
    6. 추가를 클릭합니다. 상단 목록에 입력 내용이 반영됩니다.

    ※ 보안정책은 웹 방화벽 관리콘솔에서 세부 설정이 가능합니다.

    ※ 보안 정책 수준에 의한 세부적인 탐지 룰에 대한 이슈 및 탐지 로그를 확인하는 방법은 관리콘솔에 내장된 매뉴얼을 참고하시기 바랍니다.

    ※ 로드밸런서를 같이 이용하는 경우 탐지 로그에서 Client IP를 확인할 수 있도록 웹서버와 LB에서 X-Forward-For 옵션 이용을 권장합니다.
